eduQuery examples

Below you'll find a description, lists of requirements and options and one or more interative examples of each eduQuery component. Opening any of the interactive examples will also provide you with a link to a text write-up which you can use to get started writing the component into your lesson/course.

Interactions

Matching

Match statements/words/etc with categories. Learners get a feedback message after every successful and unsuccessful match.

Requirements

  • Every "statement" needs to have a matching "answer".

Options

  • Each "statement" can have two custom messages: one for when it’s successfully matched and one for every time it’s unsuccessfully matched.
  • Custom message for when learner completes interaction.

Interactive example

  • Matching

Fill in the blank

Match words with sentence-blanks. Learners get a feedback message after every successful and unsuccessful match.

Requirements

  • Each "sentence" need to have at least one "blank".

Options

  • Each "word" can have two custom messages: one for when it’s successfully matched and one for every time it’s unsuccessfully matched.
  • "Sentences" can have more than one blank.
  • Custom message for when learner completes interaction.

Interactive example

  • Fill in the blank

Multiple-choice

One correct answer per question. Customizable feedback for correct and incorrect answers, and finishing the interaction.

Requirements

  • Each question has only one correct answer.

Options

  • You can choose to show all questions to the learner or just one at a time.
  • Each answer can have a custom message for when it’s selected by the learner.
  • Custom message for when learner completes interaction.

Interactive examples

  • One question at a time
  • View all questions

Scenario

A two part interaction: scenario and response. The user reads a scenario or story on the first page and then answers a small number of questions on the second page via Multiple-choice.

Requirements

  • Scenario text - recommended max of two paragraphs.
  • Small number of questions based on the scenario.

Options

  • Image to accompany scenario.

Interactive example

  • With Multiple-choice

Slide Scroller

Navigate through multiple slides of content.

Requirements

  • Specify what content you want on every slide.

Options

  • Content options
    • text
    • links
    • images
    • captioned voiceclip
    • single multiple-choice question

Interactive example

  • Slide Scroller

Content layering options

Show More

Simple text reveal with related image.

Requirements

  • Title that will be visible at all times.
  • Text for each show more.
  • Specify related image.

Options

  • Each Showmore can have multiple items and each item can contain: text and links.

Interactive example

  • Show More

Tabbed Info

Simple layered content.

Requirements

  • Title that will be visible at all times.
  • Text for each tabbed info.

Options

  • Each tab content can have: text, links, images.

Interactive example

  • Tabbed Info

Timeline

Clickable dates/events along a timeline.

Requirements

  • Provide a linear sequence of events.
  • Descriptive content for each event.

Options

  • In addition to text each event can have an image and/or link.

  • Timeline

Layout

PullQuote

The pullquote is used to off-set and seperate a simple textblock or a textblock with a photo.

Requirements

  • Provide the text that you would like to off-set. We recommend a small amount of text.

Options

  • In addition to text you can have an image and/or a link.

  • Pullquote

Media presentation options

Captioned Voiceclips

Voiceclips with the sort of timed captions you typically see with video.

Requirements

  • Script of exactly what you want the person to say.
  • If you have a specific type of person in mind, please provide a general description (age, gender, cultural background, etc.).

Options

  • You can have multiple voiceclips (recommended maximum of 3-4).

Interactive example

  • Captioned Voiceclips

Captioned Video

This shows how we're presenting captioned video.

Requirements

  • Script of exactly what you want the actors to say.
  • Description of location(s) where you want the video to take place.
  • If you have a specific type of person(s) in mind, please provide a general description (age, gender, cultural background, etc.).
  • Make a notation if the same actors need to be used in other course videos.
